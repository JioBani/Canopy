# Canopy — 이슈 트래커 기획서

> **제품명**: **Canopy** (캐노피) — 나무(계층 트리) 위에서 숲 전체를 내려다보는 가시성을 상징.
> **목적**: Jira 를 대체할 가벼운 자체 프로젝트 관리 웹앱의 기획 문서.
> 구현 멤버가 이 문서만 읽고 전체 기능을 파악·구현할 수 있도록 작성한다.
> (작성: main / 구현: impl 멤버, 바이브 코딩)

---

## 1. 배경과 목표

### 왜 만드는가
현재 게임 개발 프로젝트(Tatica Defense)를 Jira 로 관리 중인데:
- Jira 는 기능이 너무 방대하고 가시성이 나쁘다.
- 특히 **계층이 Epic→Task→Sub-task 3단으로 고정**이라, 우리 기획 구조(컨텐츠→기능→세부기능→…)를 담지 못해 **컨텐츠+기능을 Epic 제목에 욱여넣는** 우회를 쓰고 있다.

### 핵심 가치 (이 도구가 반드시 지킬 것)
1. **한눈에 보는 가시성** — 깊은 계층의 진행 상황을 한 화면에서 파악.
2. **빠르고 단순한 입력** — 클릭 몇 번/단축키로 이슈 생성·수정.
3. **꼭 필요한 필드만** — 타입별로 필요한 필드만 노출.
4. **유연하지만 일관된 계층** — Jira 보다 깊은 트리, 단 타입 규칙은 엄격히 강제.

### 설계 철학
> **기능은 Jira 를 따라하되(기본 기능만), UI/UX 는 가시성을 최우선으로 새로 설계한다.**

## 3. 핵심 개념과 용어

이 도구의 데이터는 게임 기획의 **두 축**을 담는다.

### 축 1 — 기획 트리 (무엇을 만드나)
```
프로젝트
 └ 컨텐츠          게임의 큰 영역           예) 전장
    └ 기능          기능 묶음                예) 소환수
       ├ 세부기능     기능적 동작              예) 소환수 합성
       └ 마스터데이터  구체 인스턴스            예) 캐릭터 "아리아"
          └ 작업       실제 작업 단위(=Jira Sub-task)
```

- **세부기능** 과 **마스터데이터** 는 **같은 계층(tier)** 이며 둘 다 기능의 자식이다.
  - 세부기능 = "기능적 동작" (소환수 합성 같은 동사).
  - 마스터데이터 = "테두리 안을 채우는 구체 인스턴스" (캐릭터·몬스터·스테이지·카드 등). 보통 `소환수 마스터데이터 제작` 같은 평범한 기능 노드 아래에 잎으로 나열한다.
- **작업** = 트리의 잎. Jira 의 Sub-task 에 해당. 도메인 태그가 붙는 실제 작업 단위.

### 축 2 — 작업 도메인 (어느 직군의 일인가)
**작업** 노드에는 도메인 태그가 하나 붙는다:

| 도메인 | 내용 |
|---|---|
| 기획 | 요소의 스탯·스킬·컨셉 설계 |
| 디자인 | UI·배치·애니메이션 등 "어떻게 보일지" |
| 사운드 | 모든 게임 사운드 |
| 구현 | 실제 개발 (= 이 도메인의 작업 요구사항을 *시스템 요구사항(SR)* 이라 부름) |
| 밸런싱 | 수치 밸런싱 |
| 기타 | 놓친 것들 |

### 사용자 요구사항(UR)과 작업 요구사항
- **UR (사용자 요구사항)**: 플레이어/운영자 시점의 요구를 도메인 언어로 적은 것. "무엇/왜"만 결정.
- **작업 요구사항**: UR 을 만족시키려면 무슨 작업을 해야 하는지 나열한 것. 한 번에 작업하기 좋은 단위로 묶은 것이 곧 **작업(Sub-task)**.
- UR 은 트리 노드가 아니라 **기능이 소유하는 그룹화된 목록**이며, **작업과 M:N 링크**로 연결된다 (§6).

## 5. 노드 타입 문법 (엄격 강제)

앱은 아래 부모-자식 규칙을 **강제**한다. 잘못된 위치에는 노드를 만들 수 없게 막는다.

| 노드 타입 | 부모로 가능 | 자식으로 가능 |
|---|---|---|
| 컨텐츠 | (없음 / 최상위) | 기능 |
| 기능 | 컨텐츠 | 세부기능, 마스터데이터 |
| 세부기능 | 기능 | 작업 |
| 마스터데이터 | 기능 | 작업 |
| 작업 | 세부기능, 마스터데이터 | (없음, 잎) |

**현재 규칙 (확정):**
- 컨텐츠 중첩 없음 (컨텐츠 밑에 컨텐츠 X).
- 기능 중첩 없음 (기능은 전부 컨텐츠 직속).
- 작업 중첩 없음 (작업은 잎).

## 6. UR 과 추적성

### UR 관리
- UR 은 **기능 노드**에서 나열·관리한다 (설명란과 별개의 구조화된 목록).
- 가독성을 위해 **UR 그룹**으로 묶을 수 있다 (한 기능에 UR 이 많이 쌓이므로).

### UR ↔ 작업 링크
- 작업 상세에서 "이 작업이 만족시키는 UR" 을 연결한다 (M:N).
- 작업 화면에서는 **작업내용 체크리스트와 연결된 UR 을 나란히** 보여줘, 사람이 매핑을 한눈에 보게 한다.

### UR 진행도 (중요)
> 산출물 진행도는 본래 **UR 단위**로 추적되는 게 가장 의미 있다.
- UR 의 진행도 = **그 UR 에 연결된 작업들의 완료 상태 집계**.
  - 예: UR "2마리 합성→상위" 에 작업 2개가 링크 → 둘 다 완료 카테고리면 "2/2 완료".
- 기능의 UR 목록에서 각 UR 옆에 `[연결 작업 n개 / 완료 m개]` 커버리지를 표시한다.
- 연결된 작업이 0개인 UR = **미커버 UR** 로 강조 (빠진 요구사항 경고).

## 7. 티켓 키 / 상태

### 티켓 키
- 프로젝트 생성 시 **키 프리픽스**를 지정 (예: `TD`).
- 노드 생성 시 프로젝트 카운터에서 번호를 받아 `TD-1`, `TD-2` … 부여. 모든 노드 타입에 발급.

### 상태 (§4 status 참고)
- 고정 카테고리 4종: **할일 / 진행중 / 완료 / 취소됨**.
- 각 카테고리 안에서 사용자가 세부 상태를 추가 (예: 진행중 → 개발중·검토중).
- roll-up·보드는 카테고리 기준.

## 8. 화면 설계 (가시성 우선)

> Jira 가 못 한 "깊은 계층을 한눈에" 를 정면으로 푼다.
> 공통: 색으로 상태 구분 · 진행바 roll-up · 타입별 아이콘/색 · 키보드 인라인 추가 · 타입별 필요한 필드만 노출.

### 8.1 메인 — 트리 + 상세 분할뷰 (핵심 화면)
좌측 계층 트리(접기/펼치기 + 각 노드 진행률 바), 우측 선택 노드 상세.

```
┌──────────────────────────────────────────────────────────────────────────┐
│ [TD] 타티카 디펜스 ▾    🔍 검색…              [트리] [보드] [대시보드]   👤 │
├──────────────────────────────────┬─────────────────────────────────────────┤
│ 트리                              │ TD-42  소환수 합성            [세부기능]  │
│ ▾ 📦 전장              ▓▓▓░ 75% │ ─────────────────────────────────────── │
│   ▾ ⚙ 소환수           ▓▓░░ 60% │ 상태 —     담당 —                        │
│     ▾ 🔧 소환수 합성    ▓▓▓░ 70% │ 설명  (마크다운 편집)                     │
│         · 합성 로직     [진행중] │                                          │
│         · 합성 UI       [할일]   │ ── 연결된 UR ────────────────            │
│     ▸ 🔧 소환수 소환    ▓░░░ 25% │  ✓ 2마리 합성→상위 소환수   [2/2 완료]   │
│   ▾ 🎴 소환수 MD 제작   ▓▓▓▓100% │  ◑ 합성 실패 처리           [1/3]        │
│       · 아리아          [완료]   │                                          │
│ ▸ 📦 정비                        │                                          │
│ [+ 컨텐츠]                        │                                          │
└──────────────────────────────────┴─────────────────────────────────────────┘
```

- 각 행의 진행바 = 하위 작업의 완료 카테고리 비율 roll-up.
- 작업 행은 상태 뱃지(색)로 즉시 구분.
- 인라인 `[+]` 로 자식 노드 빠른 추가 (타입은 문법에 따라 자동 한정).

### 8.2 작업 상세 (이슈 안에서 보이는 것)
요청하신 필드: **티켓명 · 제목 · 설명 · 요소(연결 UR·체크리스트·도메인) · 작업 상태 · 작업자.**

```
┌ TD-43  소환수 합성 UI 작업 ──────────────────────────────────┐
│ 타입 작업   도메인 [디자인]   상태 [할일 ▾]   담당 [👤 ▾]     │
│ 설명  (마크다운)                                             │
│ ── 작업내용 (체크리스트) ──         ── 만족시키는 UR ──       │
│  ☑ 합성 결과 미리보기 패널           · 2마리 합성→상위        │
│  ☐ 합성 버튼 상태 처리               · 합성 실패 처리         │
│  ☐ 실패 연출                        [+ UR 연결]              │
│ ── 선제조건(blocks) ──                                       │
│  ⛓ TD-12 스킬 시스템 인프라                                  │
└──────────────────────────────────────────────────────────────┘
```

### 8.3 보드 뷰 (칸반)
작업 실행 추적용. 기본 컬럼 = 4 카테고리. 필터로 좁힘.

```
┌ 보드 — 필터: [도메인▾] [담당자▾] [컨텐츠▾] [기능▾] ───────────────┐
```

- 카드 드래그로 상태 변경. 카드엔 티켓키·제목·도메인·담당자.

### 8.4 기능 / UR 추적 뷰
기능 노드를 열면 UR 목록(그룹화) + 각 UR 커버리지. 미커버 UR 강조.

```
┌ ⚙ 소환수 — 사용자 요구사항 ──────────────────────────────────┐
│ 그룹: 합성                                          [+ UR]    │
│   ✓ 2마리를 합쳐 상위 소환수를 얻는다     작업 2 / 완료 2      │
│   ◑ 합성 실패 시 재료를 보존한다          작업 3 / 완료 1      │
│   ⚠ 합성 확률을 표시한다                  작업 0 (미커버!)     │
│ 그룹: 육성                                          [+ UR]    │
│   …                                                          │
└────────────────────────────────────────────────────────────────┘
```

### 8.5 대시보드
프로젝트 한눈 요약: 컨텐츠별 진행률, 도메인별 작업 분포, 내 작업(담당자=나), 미커버 UR 수.

### 8.6 프로젝트 선택 / 생성
- 상단 프로젝트 스위처(▾)로 전환.
- 생성 모달: 이름 + **키 프리픽스** 입력. 기본 상태 셋(4 카테고리 × 기본 1개) 자동 시드.

## 10. 비기능 / UX 원칙 (가시성)

1. **상태는 색으로** — 카테고리별 색 일관 적용 (할일=회색, 진행중=파랑, 완료=초록, 취소=빨강/흐림).
2. **roll-up 진행바** — 모든 비-잎 노드에 하위 완료율 표시.
3. **타입별 최소 필드** — 컨텐츠는 거의 제목만, 작업만 도메인·담당자·체크리스트.
4. **빠른 입력** — 인라인 추가, 키보드 우선, 모달 최소화.
5. **컨텍스트 유지** — 트리+상세 한 화면, 페이지 이동 최소.
6. **밀도 조절** — compact / comfortable 토글.
7. **미커버 UR·막힌 작업 강조** — 빠진 것/막힌 것이 눈에 띄게.

## 부록 — 한 줄 요약
**프로젝트 > 컨텐츠 > 기능 > (세부기능|마스터데이터) > 작업** 의 엄격한 타입 트리.

상태는 4 카테고리(할일/진행중/완료/취소됨) 안에서 커스텀. 진행률은 카테고리 기준 roll-up.
기능은 Jira 기본기, UI 는 가시성 최우선. Vite/React/Tailwind/shadcn + Supabase + Vercel, 전부 무료.
