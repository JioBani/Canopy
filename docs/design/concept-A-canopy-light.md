# Canopy 디자인 컨셉안 — A안 "빛이 닿는 나무"

> 출처: design 멤버가 `frontend-design` 스킬 관점으로 도출한 0→1 기초 미감 방향.
> 상태: **제안(draft)**. 코드 적용은 main dispatch 후 impl 이 수행.
> 대안 안들과 비교용 — 다른 안은 `concept-B-*.md` 등으로 추가.

---

## 1. 테제 (한 문장)
**"캐노피 위에서 내려다보면, 빛이 닿은 곳이 다 자란 곳이다."**

계층 트리를 파일 탐색기가 아니라 *숲의 단면*으로 본다. 진행/상태를 **숲을 통과하는 햇빛**으로
렌더한다 — 그늘(할일) → 어룽진 빛(진행중) → 완전한 햇빛(완료). 깊은 계층에서도 "어디까지
빛이 닿았나"가 한눈에 보이는 것, 그게 Canopy 의 존재 이유이자 시그니처다.

레퍼런스 톤: 토스/배민 — 화려함이 아니라 *여백 많은 차분한 캔버스 + 확신에 찬 단일 액센트 +
흠잡을 데 없는 간격/타이포*.

## 2. 컬러 토큰 (4–6 named)
무채색 shadcn 기본값을 버리고, 밝고 차분한 캔버스 + 단일 그린 액센트로 간다.
토스의 "하나의 확신찬 블루"를 Canopy 는 그린으로 한다.

| 이름 | 역할 | HEX | OKLCH(근사, shadcn `index.css` 용) |
|---|---|---|---|
| **Mist** | 캔버스 배경 (순백 아님, 빛 머금은 거의-흰색) | `#FBFCFA` | `0.992 0.004 150` |
| **Bark** | 잉크/전경 (순검정 아님, 나무껍질 먹색) | `#1B2420` | `0.23 0.012 155` |
| **Canopy** | 브랜드/primary/완료 (자연 잎-그린, 애시드 아님) | `#1E8E5A` | `0.58 0.12 158` |
| **Sun** | 진행중/활성 하이라이트 (어룽진 햇빛, 꿀-앰버) | `#E0A12E` | `0.76 0.14 75` |
| **Stone** | 할일/muted (초록기 도는 회색) | `#8A938D` | `0.64 0.012 155` |
| **Ember** | 위험/미커버 UR/막힌 작업 (떨어진 잎·꺾인 가지) | `#E2533B` | `0.62 0.18 32` |

> **핵심 결정 — 상태 램프를 "빛"으로**: 할일(Stone, 그늘) → 진행중(Sun, 어룽빛) →
> 완료(Canopy, 햇빛) → 취소(Bark 흐림, 낙엽). 기획서 §10 은 "진행중=파랑"이지만, 파랑은
> Canopy 의 숲 세계관과 무관한 관습값이다. 이걸 햇빛 램프로 바꾸는 게 **시그니처를 성립시키는
> 한 번의 리스크**다.
> **관습 안전판**: 합의 안 되면 진행중만 블루로 회귀 가능 — 단 시그니처는 약해진다.

## 3. 타이포
- **본문/UI: Pretendard** — 한국어 프로덕트급 표준 페이스. Inter(AI 디폴트·금지) 대신
  *한국어 도구 정체성*에 맞는 선택. 8시간 쓰는 도구라 디스플레이 폰트로 피로 주지 않고,
  개성은 컬러/간격/시그니처가 진다.
- **데이터/티켓키: JetBrains Mono** — `TD-42`, 진행률 %, tabular 숫자. 트래커에서 티켓키는
  도처에 있으니 확신찬 모노로 "데이터" 인격을 부여.
- **위계**: Pretendard 800(타이트 트래킹)으로 헤더/브랜드 — 둘째 폰트 없이 편집적 자신감.
  (샤넬 룰 — 액세서리 하나 빼기: 디스플레이 폰트 추가 안 함.)

## 4. 레이아웃 & 시그니처
보들은 트리+상세 분할(기획서 8.1). **시그니처는 트리에 산다** — Canopy 의 본분이니까.

핵심 장치 **"빛이 닿는 나무"**:
- 들여쓰기 가이드선 = *가지*. 회색 직선이 아니라 **하위 완료율에 따라 그린으로 물드는**
  연결선. 다 자란 가지(subtree 완료)는 햇빛색으로 빛남.
- 선택 노드까지의 root→node 경로가 *빛나는 길*로 강조 → 컨텍스트 유지.
- 타입색을 무지개(현재 amber/sky/violet/emerald = "랜덤색 배정" 티)에서 **숲 단면의 톤 램프**로:
  컨텐츠(우듬지 그린) → 기능(나무껍질) → 세부기능/마스터데이터(두 잎-톤). 조화롭게.

```
┌ Canopy ─ [TD] 타티카 디펜스 ▾      🔍               트리 · 보드 · 대시보드  👤 ┐
├──────────────────────────────┬──────────────────────────────────────────────┤
│ ▾ ◆ 전장            ███▓ 75% │  TD-42                              세부기능  │
│ ┃ ▾ ◆ 소환수         ██▓░ 60% │  소환수 합성                                  │
│ ┃ ┃ ▾ ◆ 소환수 합성  ███▓ 70% │  ───────────────────────────────────────     │
│ ┃ ┃ ┃· 합성 로직   ◑ 진행중 │  상태 ◑진행중   도메인 디자인   담당 ◷ 아리아 │
│ ┃ ┃ ┗· 합성 UI     ○ 할일   │                                              │
│ ┃ ┗ ▸ ◆ 소환수 소환  █░░░ 25% │  설명 …                                       │
│ ┗ ▾ ◆ MD 제작      ████100% │  ── 연결된 UR ────────                        │
│     ┗· 아리아     ● 완료   │   ● 2마리 합성→상위        2/2 ███████        │
│ ▸ ◆ 정비                    │   ◑ 합성 실패 처리         1/3 ███░░░░        │
│  ↑ 가지선이 완료율 따라 그린으로 물듦 (빛이 닿음)                            │
└──────────────────────────────┴──────────────────────────────────────────────┘
```
- 보드(칸반): 4컬럼 헤더에 햇빛 램프 색(그늘→빛). 카드는 조용히.
- **첫인상 = 빈 상태**: 현재 "아직 컨텐츠가 없습니다"는 밋밋함. 여기에 컨셉을 심는다 →
  *"첫 나무를 심으세요"* + 작은 캐노피 일러스트. 시그니처가 처음 만나는 순간.

## 5. 규율 (대담함은 한 곳에만)
- 보들 트리만 빛난다. 상세패널·폼·뱃지는 **토스처럼 조용히**: 넉넉한 여백, 단일 액센트,
  차분한 표면.
- 품질 바닥(말없이 충족): 모바일 반응형, 키보드 포커스 링(Canopy 그린),
  `prefers-reduced-motion` 시 빛-애니메이션 정지.
- 카피: 시스템 언어 금지 → "삭제됨"이 아니라 행동 중심. 빈 화면은 초대장으로.
