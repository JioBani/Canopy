# 상세 커버 헤더 + 제목 위계 사양 (impl 적용용)

> 대상: impl. Pixel Blossom + 메이플 2-tier 위. 코드는 이 문서 기준.
> 목적(사용자 페인): "지금 내가 **어느 레이어의 어디에** 있는지" 가 한눈에 안 보임 → 노션 커버처럼
> 상단 헤더가 **레이어(색) + 위치(큰 브레드크럼)** 를 책임지고, **제목은 깔끔하게**.
> (Jira 가 못 해서 제목에 `[전장-유닛]` 을 욱여넣던 트라우마 — 도구가 대신 해결.)

---

## 0. 한 줄 요약
모든 노드 상세 상단에 **레이어-틴트 커버 밴드**: `레이어 칩(색+아이콘) + 큰 브레드크럼 + 티켓키`. 밴드 아래 **큰 제목(메이플 28~32px)**. 레이어 5색은 상태색·도메인칩과 분리된 별도 의미축, **커버에만** 등장(한 화면 한 색 → 무지개 방지).

---

## 1. 커버 헤더 구조

```
┌─ 커버 밴드 (레이어 틴트, 상단 라운드, 좌측 3px 레이어 액센트) ───────────────┐
│  ▢ 세부기능                                                  TACO-42  [복사]  │
│  전장  ›  유닛                                            (브레드크럼 — 크게)  │
└────────────────────────────────────────────────────────────────────────────────┘
  소환수 합성                                  ← 제목 30px 메이플 Bold (밴드 아래)
  [ 상태 ◑진행중  도메인 디자인  담당 ◷아리아 ]   (property — 작업만)
  ▸ 설명 …
```

- **밴드**: 상세 본문 상단 풀폭. bg = `레이어색 12%` 틴트(`color-mix(in oklab, var(--layer-x) 12%, white)`), 상단 좌우 라운드 14px, **좌측 3px 레이어색 액센트**(또는 상단 3px). 높이 auto(~64px, 2줄). 헤더만 색, 본문은 흰색 유지.
- **레이어 칩**(좌상): 픽셀 타입 아이콘(레이어색) + 레이어명 "세부기능"(레이어 ink, 13px bold). → "이건 OO 레이어" 즉시 인식.
- **브레드크럼**(좌하, **크게**): `전장 › 유닛` (조상 경로만, 현재 노드는 아래 제목). **14px / medium / ink-2**, 세그먼트 클릭 → 해당 노드로 점프(드릴업). `›` = ink-3. (현재 12px 회색 → 또렷하게 키움)
- **티켓키 + 복사**(우상): plum mono 12px + ghost 복사.
- **제목**(밴드 아래): **메이플 Bold, clamp(28px,4vw,32px)**, ink(#272129), leading 1.2. **레이어색 안 씀**(깔끔). 편집형 Input 유지(hover 만 미세 bg). 좌측에 얇은 레이어색 4px 바(선택, 위계 보강).

> 위치(브레드크럼)+레이어(색·칩)는 커버가, 제목은 내용만 — 역할 분리.

## 2. 레이어 색 시스템 (Pixel Blossom "정원의 레이어")

5개 레이어를 **저채도 distinct 색**으로. 한 화면에 **커버 한 곳만** 등장 → 무지개 안 됨. 상태(mist/peach/sakura/ember)·도메인칩(pink-bg/plum)과 **다른 의미축**이라 충돌 없음.

| 레이어 | 테마 | base(아이콘·틴트·액센트) | ink(칩 라벨 텍스트, AA) |
|---|---|---|---|
| **컨텐츠** | 장미 Rose | `#E27BA4` | `#B84A77` |
| **기능** | 라벤더 Lavender | `#A892D9` | `#6E55B0` |
| **세부기능** | 수국 Sky | `#7DAEDE` | `#3F6FA6` |
| **마스터데이터** | 민트 Mint | `#5FBBA2` | `#2E7E6A` |
| **작업** | 금잔화 Honey | `#DDA24A` | `#9C6B1C` |

```ts
// src/nodes/layerColors.ts (신규)
export const LAYER_COLOR: Record<NodeType, { base: string; ink: string }> = {
  컨텐츠:      { base: "#E27BA4", ink: "#B84A77" },
  기능:        { base: "#A892D9", ink: "#6E55B0" },
  세부기능:    { base: "#7DAEDE", ink: "#3F6FA6" },
  마스터데이터:{ base: "#5FBBA2", ink: "#2E7E6A" },
  작업:        { base: "#DDA24A", ink: "#9C6B1C" },
}
```
- **사용처(절제)**: 커버 밴드 틴트·액센트, 레이어 칩(아이콘 base + 라벨 ink), 제목 좌측 액센트 바(선택). **그 외엔 안 씀.**
- **선택 강화(권장)**: 트리/브레드크럼의 **타입 아이콘**에 레이어 base 색 적용 → 트리에서도 레이어 인식 일관. 단 밀집 트리라 **아이콘에만**(텍스트·배경 색칠 금지)로 절제. (현재 컨텐츠만 sakura → 5색 체계로 교체)
- 기존 `TYPE_META.color`(amber/sky/violet/emerald 잔재 있으면)와 `nodeGrammar`/`pixelIcons` 색 운용을 이 표로 **단일화**.

---

## 3. 브레드크럼 (크게)
- 조상 경로만(루트→부모). 현재 노드 제외(제목이 담당).
- **14px / font-medium / ink-2**, 세그먼트 hover → ink, 클릭 → openNode(점프). `›`(또는 `/`) 구분자 ink-3, gap 6px.
- 길면 중간 생략(`전장 › … › 소환수`)으로 한 줄 유지 + 전체는 Tooltip.
- 컨텐츠(부모 없음): 브레드크럼 생략, 커버는 레이어 칩+제목만.

---

## 4. 제목 위계
- **제목**: 메이플 Bold, **clamp(28px, 4vw, 32px)**(현 21px → 키움), tracking -0.01em, leading 1.2, ink.
- 위계: 레이어 칩 13px < 브레드크럼 14px < **제목 30px**. 커버 틴트가 제목을 받쳐 위계 강화.
- 제목은 색 안 입힘(깔끔). 위계·정체성은 커버가.

---

## 5. 타입별 적용 (전 레이어 공통)
모든 상세(컨텐츠~작업)에 동일 커버 적용 — 어디서든 "레이어+위치" 인식.
- 컨텐츠: Rose 커버, 브레드크럼 없음.
- 기능: Lavender, `전장`.
- 세부기능: Sky, `전장 › 유닛`.
- 마스터데이터: Mint, `전장 › 유닛`.
- 작업: Honey, `전장 › 유닛 › 소환수 합성`. (작업 상세 페이지 = 커버 + 세로 스택, [spec-subfeature-panel.md](spec-subfeature-panel.md) §3-7 와 합치)

---

## 6. 접근성 / 대비
- 레이어 칩 라벨 = **ink 변형**(위 표) → 틴트/흰 위 AA 충족. base 는 아이콘·틴트·굵은 액센트에만(작은 텍스트 금지).
- 색만으로 레이어 구분 안 함 — **픽셀 타입 아이콘 + 레이어명 텍스트** 동반(색맹 대응).
- 브레드크럼/제목 대비 충분(ink-2/ink). `prefers-reduced-motion` 무관(정적).

## 부록 — 색 충돌 점검
- 상태(개화): mist/peach/sakura/ember — 작은 글리프, property/리스트. **레이어색과 위치·크기 달라 안 겹침.**
- 도메인 칩: pink-bg 배경+plum. 커버(레이어 틴트)와 다른 영역.
- 레이어 작업=Honey(#DDA24A) vs 진행중=peach(#EC9A78): 톤 비슷하나 **맥락 분리**(커버 밴드 vs 상태 글리프). 우려되면 작업 레이어를 더 또렷한 Honey ink 로 칩 표시.
