## 1. 테제 (한 문장)
**"트리는 벚꽃나무다 — 일이 끝나면 가지에 꽃이 핀다."**

Canopy 는 *게임을 키우는 도구*다. 픽셀아트는 게임 제작의 모국어이므로, 카와이 도트는 장식이
아니라 **소재의 정직한 반영**이다. 계층 트리를 부드러운 픽셀 벚꽃나무로 보고, 진행/완료를
**꽃이 피는 것**으로 렌더한다 — 닫힌 꽃봉오리(할일) → 반쯤 핀 봉오리(진행중) → 만개(완료).
A안의 "성장/빛" 테제를 "개화"로 옮긴 것이라 두 안은 사촌지간.

레퍼런스 무드: 딸기우유 + 벚꽃, 서브컬쳐 카와이. **부드럽고 저채도, 눈에 잘 익는** 색감.

## 2. 컬러 토큰 (저채도 파스텔)
"너무 쨍하지 않게"가 핵심 제약 → 채도 25~55%, 밝은 캔버스, 잉크는 순검정 대신 **부드러운
자두-코코아색**. 작은 텍스트엔 대비 확보용 진한 변형(Plum)을 따로 둠.

|---|---|---|---|
| **Milk** | 캔버스 배경 (딸기우유 크림) | `#FFF5F7` | `0.974 0.012 350` |
| **Cloud** | 카드/표면 (거의-흰, 보드라움) | `#FFFBFC` | `0.992 0.004 350` |
| **Cocoa** | 잉크/전경 (순검정 아님, 부드러운 자두-코코아) | `#5A4A52` | `0.42 0.02 350` |

> **핵심 결정 — 상태 램프를 "개화"로**: 할일(Mist, 닫힌 봉오리) → 진행중(Peach, 반개) →
> 완료(Sakura, 만개) → 취소(시든, 저채도 흐림). 기획서 §10 "진행중=파랑"에서 벗어나는 게
> **시그니처를 성립시키는 한 번의 리스크**(A안과 동일 구조). 파랑(Sky)은 보조/선택 강조로 빠짐.
> **관습 안전판**: 합의 안 되면 진행중=Sky 블루로 회귀 가능.

## 3. 타이포 — 핵심은 "픽셀은 악센트에만"
8시간 보는 밀도 도구라 본문에 픽셀 폰트를 쓰면 가독성이 죽는다. 그래서 분리:
- **본문/UI: Pretendard** — 한국어 가독성 유지. 빽빽한 트리·폼·설명 다 여기로.
- **픽셀 악센트: Galmuri (갈무리)** — 한국어 비트맵 폰트, 인디·서브컬쳐 게임 씬의 정본.
  **브랜드 워드마크 "Canopy", 티켓키(`TD-42`), 큰 숫자/진행률, 섹션 eyebrow** 에만 사용.
  영문 보조가 필요하면 Silkscreen / DungGeunMo.
- 즉 개성(픽셀·카와이)은 chrome 과 데이터에만, 콘텐츠는 깔끔. (샤넬 룰 — 픽셀은 한 겹만.)

## 4. 레이아웃 & 시그니처
보들은 트리+상세 분할(기획서 8.1). **부드러운 둥근 카드(컨테이너) + 픽셀 스프라이트(내용물)** —
레퍼런스처럼 표면은 보드랍게, 픽셀은 아이콘·진행바·장식에.

핵심 장치 **"피는 벚꽃나무"**:
- **노드 타입 아이콘 = 16px 픽셀 스프라이트** (현재 lucide 라인 아이콘 교체):
  컨텐츠=픽셀 나무, 기능=픽셀 톱니, 세부기능=픽셀 공구, 마스터데이터=픽셀 카드, 작업=픽셀 체크박스.
  게임 소재 + 기능 구분 동시 충족.
- **진행바 = 청크 픽셀 블록** (레트로 EXP/HP 바): `▰▰▰▱▱` 만개핑크 블록이 차오름. 매끈한 바보다
  테마에 맞고 칸 수로 더 잘 읽힘.
- **개화 시그니처**: subtree 완료율이 오르면 그 가지에 작은 픽셀 벚꽃이 핀다(꽃 스프라이트/가지 핑크 물듦).
- **빈 상태 = 첫인상**: 헐벗은 픽셀 묘목 + "첫 씨앗을 심어보세요". 노드를 채울수록 꽃이 핌.
- **완료 셀레브레이션**: 작업이 완료로 바뀌는 *순간에만* 작은 꽃잎 퍼프(상시 앰비언트 아님 — 산만/AI-slop 방지).

```

├──────────────────────────────┬──────────────────────────────────────────────┤
│ ▾ 🌳전장            ▰▰▰▱ 75% │  TD-42  (Galmuri)                  세부기능   │
│  ▾ ⚙소환수          ▰▰▱▱ 60% │  소환수 합성                                  │
│   ▾ 🔧소환수 합성    ▰▰▰▱ 70% │  ─────────────────────────────────           │
│    ✿합성 로직      ◐ 진행중 │  상태 ◐진행중  도메인 디자인  담당 ◷아리아   │
│    ❀합성 UI        ○ 할일   │  설명 …                                       │
│   ▸ 🔧소환수 소환    ▰░░░ 25% │  ── 연결된 UR ────────                        │
│  ▾ 🎴MD 제작        ▰▰▰▰100%│   ✿ 2마리 합성→상위        2/2 ▰▰▰▰▰         │
│    ✿아리아         ● 완료   │   ◐ 합성 실패 처리         1/3 ▰▱▱▱▱         │

```
- 보드(칸반): 4컬럼 헤더에 개화 램프 색(봉오리→만개). 카드는 둥근 Cloud 표면.

## 5. 규율 (부드럽게, 한 곳만 대담하게)
- **파스텔은 캔버스, 픽셀 카와이는 시그니처에만.** 상세패널·폼·설명은 Cloud 흰 표면 + Pretendard
  로 조용하고 읽기 쉽게. 핑크 떡칠 금지.
- 저채도 유지로 "쨍함" 차단. 잉크는 Cocoa(자두-코코아)로 부드럽게. 작은 텍스트는 Plum 으로 대비 확보(AA).
- 품질 바닥: 모바일 반응형, 키보드 포커스 링(Sky/Plum), `prefers-reduced-motion` 시 꽃잎/개화 모션 정지.
- 카피: 카와이라도 시스템 언어 금지·행동 중심. 빈 화면은 "심어보세요" 초대장으로.

## 6. A안과의 관계
- **공통 시그니처 구조**: "성장의 가시화" (A=빛이 닿음 / B=꽃이 핌). 상태 램프 리스크도 동일.
- **차이**: A=차분한 그린·토스/배민 톤(전문 도구 인상). B=딸기우유 파스텔·서브컬쳐 카와이(친근·게임팀 정서).
- 둘은 **토큰만 갈아끼우면 전환 가능**한 한 뼈대 위에 설계됨 → 프로토타입 후 택1 또는 절충 가능.
