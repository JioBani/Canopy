# 세부기능 패널 + 사이드바 UX 사양 (impl 적용용) — 확정본

> 대상: impl. Pixel Blossom + 메이플 2-tier 디자인시스템 위에서 구현.
> 상태: **확정**(main 레이아웃 결정 반영). 코드는 이 문서 기준.
> 디자인 토큰·컴포넌트는 [design-system-B-handoff.md](design-system-B-handoff.md) + 현재 코드(shadcn) 따른다.

---

## 0. 한 줄 요약
세부기능의 **UR/작업을 두 곳에** 둔다 — **좌측 트리 사이드바**(항상 접힘·컴팩트·네비)와 **우측 상세 패널**(풀 편집 surface). 둘은 형제(N:M, 작업은 UR 하위 아님).
UR은 그룹별 + **3상태(완료/미구현/오구현, 개화글리프와 분리한 원/경고 언어)**, 작업은 카드별 체크리스트 + 만족 UR. 긴 내용은 접힘+말줄임+펼침, 사이드바는 드래그 리사이즈.

## 1. 레이아웃 — 듀얼 배치

```
┌ 트리 사이드바 (리사이즈) ─║◀핸들▶─ 세부기능 상세 (풀 편집) ────────────────────┐
│ ▾ 🌳 전장               ║  전장 / 소환수                        (브레드크럼)     │
│  ▾ ⚙ 소환수             ║  TACO-42 [복사]                        세부기능        │
│   ▾ 🔧 소환수 합성       ║  소환수 합성                           (제목·메이플)   │
│     ┌ [요구사항12│작업8]┐║  ▸ 설명 (접힘)                                        │
│     │ ▸ 합성 (그룹)     │║ ┌ [ 요구사항 12 │ 작업 8 ]   [모두 펼치기] ─────────┐ │
│     │ ▸ 육성 (그룹)     │║ │ ▾ 합성        완료5·미구현2·⚠1        [+ UR]      │ │
│     └──────────────────┘║ │   ○ 2마리 합성→상위 …            작업2/완료2  ▸    │ │
│   ▸ 🔧 소환수 소환       ║ │   ⚠ 합성 확률 고정 …             작업1        ▸    │ │
│  ▸ ⚙ 정비               ║ │ ▾ 육성 …                                          │ │
│  ↑ 사이드바: 항상 접힘   ║ └──────────────────────────────────────────────────┘ │
│    기본, 클릭 펼침       ║   ↑ 상세: 더 펼친 작업용 본문(인라인 펼침·편집)        │
└─────────────────────────╨────────────────────────────────────────────────────────┘
```

- **좌측 사이드바(트리)**: 세부기능 행 아래 UR/작업을 **임베드**. **항상 collapse 기본**, 클릭해 펼침(밀집 트리 안 어지럽히게). 컴팩트 = 읽기·네비 용도.
- **우측 상세 패널**: 세부기능 선택 시 같은 UR/작업을 **풀 편집 surface**로(인라인 펼침·전문·편집·오구현 사유).
- **일관성**: 두 곳 모두 UR 3상태 글리프·오구현 ember·그룹 헤더 동일 언어. 차이는 *밀도/편집 깊이*뿐.
- **적용 = 세부기능 ONLY.** UR 소유 기존 `기능` → **세부기능** 이동. **마스터데이터는 UR 없음**(→ 작업 직접 fan-out): 마스터데이터는 사이드바·상세 모두 **작업만**(토글/UR 없음, WorkSection 재사용).

## 2. 좌측 사이드바 — 임베드 UR/작업 (컴팩트)

세부기능 트리 행을 펼치면(▾), 들여쓰기 아래 **임베드 미니 패널**:

- **미니 세그먼트 토글** `[요구사항 n │ 작업 n]` (상세보다 작게, 높이 28px). 카운트 뱃지, 오구현 ≥1이면 요구사항 옆 ember 점.
- **선택 섹션 = 항상 접힘 기본**:
  - **요구사항**: `ur_group` 그룹 헤더만 먼저(▸ 접힘). 그룹 클릭 → UR 행 펼침. UR 행 = `[3상태 글리프] 텍스트(말줄임) · 커버리지`. 한 줄 고정.
  - **작업**: 세부기능의 작업들(잎). 행 = `[개화글리프] 티켓키 제목(말줄임) · 진행`. **행 클릭 → 그 작업 선택(상세 로드)**.
- **긴 한 줄**: 사이드바는 **말줄임 + hover Tooltip(전문)만**(인라인 wrap 펼침은 상세에서). 컴팩트 유지.
- **편집 없음**: 사이드바는 보기/네비 전용. 추가·상태변경·사유는 상세에서. (예외: 작업 선택 네비는 가능)
- 구현 노트: 세부기능 펼침은 **raw 작업 자식행 대신 이 토글 패널**로 표시(작업은 패널 작업탭에 들어감). 마스터데이터는 기존처럼 작업 자식행 그대로.

## 3. 우측 상세 패널 — 풀 편집 surface

세부기능 상세 = **설명 + [요구사항 | 작업] 토글 섹션**.

### 3-1. 설명
세부기능도 설명 보유. 길 수 있어 **기본 접힘**(1줄 미리보기 + ▸), 펼치면 마크다운 전체.

### 3-2. UR↔작업 토글 (상세)
- 세그먼트 컨트롤 `[요구사항 12 │ 작업 8]`(현 SegmentTab 언어, 높이 34px). 활성 = `bg-card`+soft shadow.
- 카운트 뱃지, 오구현 ≥1 → 요구사항 옆 ember 점. 우측 `[모두 펼치기 ⇕]`.
- 노드별 마지막 탭 기억(세션), 기본 = 요구사항.

### 3-3. UR 섹션
**그룹 헤더(필수)** — `ur_group` 묶음, 미분류는 맨 아래(비면 숨김).
```
▾ 합성                              완료 5 · 미구현 2 · ⚠오구현 1   [+ UR]
  ○ 2마리를 합쳐 상위 소환수를 얻는다 …            작업 2/완료 2   ▸
  ⚠ 합성 확률은 10%로 고정한다 …                  작업 1          ▸
```
- 헤더: `▾/▸` 접기 + 그룹명(Pretendard 13px semibold) + **상태 롤업**(완료 n·미구현 n·⚠오구현 n; 0인 칩 생략) + `[+ UR]`. hover 시 이름변경·삭제.
- **UR 행**: `[3상태 글리프] 텍스트(말줄임) · 커버리지 · ▸`.
  - 긴 한 줄 → 말줄임 + hover Tooltip + **▸/행클릭 인라인 펼침**(전문 wrap + 연결 작업 칩 + 오구현 사유). 기본 접힘.
  - 커버리지: `작업 n / 완료 m`. 연결 0 = `미커버`(ember 강조). 기존 CoverageBadge 계승.

### 3-4. UR 3상태 글리프 — **개화글리프(작업 4상태)와 분리** (혼동 방지)
작업=꽃, **UR=원/경고**. 신규 `UR_STATE_GLYPH`.

| `ur.status` | 글리프 | 색 | 의미 |
|---|---|---|---|
| **미구현** | `○` 빈 원 | `--c-mist` | 아직 구현 안 됨(기본) |
| **완료** | `●` 체크 채운 원 | `--c-sakura`(체크 흰색) | 구현으로 충족 |
| **오구현** | `⚠` 경고 삼각 | `--c-ember` | 잘못 구현 — **강조** |

- 상태 = **수동**(저장 `ur.status` enum = 완료/미구현/오구현): 인디케이터 클릭 → 작은 메뉴(shadcn DropdownMenu)로 선택.
- **오구현 강조**: 행 좌측 2px ember 레일 + 배경 `rgba(216,95,110,.05)`. 그룹 헤더·토글 카운트에 ⚠ 전파.

### 3-5. 오구현 사유
- 오구현 전환 시 **사유 입력 권장**(필수 아님). 저장 `ur.misimpl_reason`(text nullable).
- **표시/편집 = 상세에서만**(사이드바 밖): UR 행 펼침 영역에 `⚠ 오구현 사유: …`(ember 박스) + 인라인 textarea 편집. ⚠ hover Tooltip = 사유 미리보기(보조).

### 3-6. 작업 섹션
세부기능에 직접 달린 작업들. 각 작업 = **접히는 카드**.
```
▸ ☐ TACO-43 소환수 합성 UI       [디자인] ◑진행중  ▰▰▱ 2/3
▾ ☑ TACO-41 합성 로직            [구현]  ●완료    ▰▰▰
   작업내용                          만족시키는 UR
   ☑ 합성 결과 미리보기              ○ 2마리 합성→상위
   ☐ 실패 연출                      ⚠ 합성 확률 고정    [+ UR 연결]
```
- 헤더(접힘 기본): `▸/▾` + 픽셀 체크박스 + 티켓키(plum) + 제목(말줄임) + 도메인칩 + 개화글리프 + 작업내용 진행(EXP+n/m).
- 펼침: 좌 `작업내용`(shadcn Checkbox) / 우 `만족시키는 UR`(연결 UR을 **UR 3상태 글리프**로 표시 + 검색형 Picker 연결). 현 TaskChecklist/TaskUrLinks 재사용.
- 기본 접힘 + `[모두 펼치기]`. 제목 말줄임 + hover Tooltip.

## 4. 사이드바 리사이즈
트리 사이드바 ↔ 상세 분할선 드래그(= 사이드바 넓이 조절).
- 핸들: 분할선 4px 히트영역, 평소 1px `--c-line`. hover/drag → **2px `--c-sakura`** + `col-resize`, 드래그 중 `user-select:none`.
- 범위 min 240 / max 560 / 기본 300px. **localStorage** 퍼시스트. **더블클릭 리셋**(300).
- a11y: `role="separator"` `aria-orientation="vertical"`, 포커스 가능, `←/→` 16px 단위, `aria-valuenow/min/max`.
- 구현: **shadcn Resizable**(`npx shadcn add resizable`) 권장 — 핸들만 위 비주얼 커스텀.

## 5. Collapse / Expand 정리

| 요소 | 사이드바(좌) | 상세(우) |
|---|---|---|
| 세부기능 UR/작업 | **항상 접힘**, 클릭 펼침(컴팩트) | 토글 섹션 상시 노출(풀) |
| 설명 | — | 접힘(1줄) → 전체 |
| UR 그룹 | 접힘(헤더만) → 행 | 펼침(행 보임) |
| UR 행 | 말줄임만(펼침 없음, Tooltip) | 접힘 → 전문+연결+사유 |
| 작업 카드 | 한 줄 행(네비) | 접힘(헤더) → 체크리스트+UR |

- 펼침 애니메이션 120~150ms, `prefers-reduced-motion` 시 즉시. 다수 그룹/카드 = shadcn **Collapsible**(`npx shadcn add collapsible`) 권장.

## 6. 긴 한 줄 처리 (공통)
UR 텍스트·작업 제목·작업내용 항목:
1. 기본 `truncate`(한 줄, 레이아웃 안전).
2. hover → shadcn **Tooltip** 전문(max 360px, 줄바꿈).
3. **상세에서만** ▸/행클릭 → `whitespace-normal` wrap 정독. (사이드바는 1·2만)

## 9. 데이터 (impl 확정 스키마)
- **UR = 세부기능 소유 only**(기능 → 세부기능 이동). 마스터데이터 UR 없음. `ur_group` 동일.
- `ur.status` enum = **완료 / 미구현 / 오구현**(수동). `ur.misimpl_reason` text nullable(오구현 사유).
- UR↔작업 = M:N(`ur_work_link`), **작업(그룹) 단위** 링크(체크리스트 항목 단위 금지).
- 커버리지(작업 n/완료 m) = 링크 작업의 status 카테고리 집계(기존 ur_coverage 뷰).

## 부록 — UR 3상태 vs 작업 4상태 (혼동 방지)
- **작업** = 개화글리프(꽃): 빈꽃 할일 / 반개 진행중 / 만개 완료 / 시듦 취소. (진행)
- **UR** = 원·경고: ○ 미구현 / ● 완료 / ⚠ 오구현. (구현 정합도)
- 모양 계열이 달라 한 화면 공존해도 구분. 오구현(⚠ ember)만 시각적으로 튄다.
